Zen Cash Specification and Roadmap

Specifications at Launch

Zen Protocol Specification

  1. Release date: 10PM EST, 23rd May 2017 as a fork from Zclassic
  2. Equihash hashing algorithm is memory-hard, proof-of-work mining algorithm based on the generalized Birthday Problem and Wagner's algorithm for it. Equihash was created by Alex Biryukov and Dmitry Khovratovich of the University of Luxembourg.
  3. Block reward 12.5 ZEN
  4. Block generation: 2.5 minutes
  5. Block size of 4 MB
  6. Difficulty adjustment algorithm: Digishield V3, tweaked to use the following trailing-average difficulty window: (next difficulty) = (last difficulty) x SQRT [ (150 seconds) / (last solve time)
  7. Division of each PoW block reward and transaction fees between miners and other stakeholders of Zen:
  1. 88.14% to miners
  2. 3.43% to Secure Node Operators
  3. 5% to Zen DAO
  4. 3.43% to Zen Core team
  1. Total eventual coin supply 21 million
  2. Reward halving period: every ~4 years, as per Bitcoin
  3. Shielded Transactions obscure sender, receiver, and amount on blockchain
  4. Transparent Transactions publish sender, receiver, and amount on blockchain
  5. Secure message field in z_transaction with 256 bytes of characters.
  6. Secure Nodes enable fully encrypted node and wallet communication
  7. Secure Nodes meeting requirements receive block and transaction rewards. Requirements:
  1. Unique Public IP
  2. Valid Certificate
  3. Maintain full Zen blockchain - tested by RPC
  4. Maintain 42 Zen in node wallet - tested by RPC
  5. Allow read-only RPC from Zen Node Tester server
  6. Send 0.00001 Zen z_transaction daily to Zen Node Tracker server with read only RPC username and password as well as t_address in message field.
  1. Governance by DAO, proposals, and voting every 2 months.
  1. Voting by holders of Voting Tokens purchased or earned during Proposal period.
  2. Zen collected during proposal period redistributed to Token holders who Voted.
